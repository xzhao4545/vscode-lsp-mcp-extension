# i18n Guardrail Rule

## When to Apply

Whenever you add, modify, or review any user-facing text in this project, you MUST ensure bilingual coverage.

## What Counts as User-Facing Text

- VSCode display names, descriptions, commands, views → `package.nls.json`, `package.nls.zh-cn.json`
- Notifications, status bar messages, dialogs → `l10n/bundle.l10n.json`, `l10n/bundle.l10n.zh-cn.json`
- README, CHANGELOG, documentation → both English and Chinese versions
- Non-obvious code comments → bilingual `// EN: ... // CN: ...` format

## Checklist Before Committing

1. **New display/notification text**: Add to BOTH language files with matching keys
2. **New settings**: Document in both `package.nls.json` AND `package.nls.zh-cn.json`
3. **README changes**: Add corresponding entry to `readme/README.zh-CN.md`
4. **Code comments**: Use bilingual format for non-obvious logic

## l10n File Locations

| Type | English | Chinese |
|------|---------|---------|
| Package display | `package.nls.json` | `package.nls.zh-cn.json` |
| Runtime messages | `l10n/bundle.l10n.json` | `l10n/bundle.l10n.zh-cn.json` |
| Docs | `README.md` | `readme/README.zh-CN.md` |
| Changelog | `CHANGELOG.md` | (optional) |

## How to Add Translations

1. Open `l10n/bundle.l10n.json` (English base)
2. Add or update the key-value pair
3. Open `l10n/bundle.l10n.zh-cn.json` and add the Chinese translation with the **same key**
4. For package metadata, edit `package.nls.json` and `package.nls.zh-cn.json`
